Claude Codeへの指示文（プロンプト）の書き方の基本

更新日: 2026/07/11 ・ 難易度: 初級

指示文の基本構造

Claude Codeに何かを依頼するとき、指示文（プロンプト）の書き方次第で出力の質が大きく変わる。基本的な構造は次の4つを意識すると組み立てやすい。

  1. 目的：何のために作るのか
  2. 仕様：具体的に何を作るのか（画面構成、データ項目など）
  3. 制約：使うツールやルール（デザインシステム、技術スタックなど）
  4. 確認方法：出来上がったものをどう確認するか

曖昧な指示と具体的な指示の違い

  • 曖昧な例：「かっこいいボタンを作って」
  • 具体的な例：「melta-uiのButtonコンポーネントを使って、プライマリカラーの送信ボタンを作って」

具体的な指示ほど、AIの出力のブレが少なくなる。逆に、まだ方針が固まっていない段階では、あえて曖昧に伝えて複数の提案をもらう、という使い方も有効。

一度に頼みすぎない

大きな機能を一気に頼むより、「まず骨組みだけ作る」→「動作確認する」→「細部を調整する」という順番で、小さく区切って依頼する方が、意図とのズレが起きにくく、途中で軌道修正もしやすい。

気づき

指示文は一度書いて終わりではなく、Claude Codeとのやり取りを重ねながら育てていくもの、という感覚を持つと気が楽になる。最初から完璧な指示文を目指さなくてよい。
