# AI Study Archive - 実装依頼

## 概要
AI学習の記録を蓄積・閲覧するための個人用アーカイブサイト。
「見返す（閲覧）」がメインの使い方で、記録の追加は随時・少しずつ行う。

- サイト名: **AI Study Archive**
- 用途: 自分（Shimi）のAIスタディの記録を蓄積し、後から見返す
- 更新頻度: 閲覧がメイン、追加はたまに

## 技術スタック
- **Astro**（静的サイトジェネレーター。Markdownファイルを自動でページ化する）
- **Tailwind CSS**
- **melta-ui**（デザインシステム。このプロジェクトルートに配置済みのリポジトリを参照し、`DESIGN.md` / `CLAUDE.md` のルールに従ってUIを実装すること）
- データ管理: サーバー・データベースなし。記事は `content/` フォルダ内のMarkdownファイルで管理（Astro Content Collectionsを使用）

## デザイン方針
- melta-uiのサンプル「bunraku-cms」（ https://melta.tsubotax.com/examples/bunraku-cms ）のダッシュボードデザイン（左サイドバー＋一覧表示のCMS風レイアウト）を踏襲する
- melta-uiのコンポーネント・トークンをそのまま使用し、独自のスタイルを追加しない

## サイト構成（左サイドバーメニュー）

1. **記事一覧**（トップページ）
2. **カテゴリ**
3. **メディア**
4. **用語集**
5. **FAQ**
6. **設定**

### 1. 記事一覧
AIスタディの記録をテーブル形式で一覧表示する。

表示項目:
- タイトル
- カテゴリ（AIツール名・手法など）
- 更新日
- ステータス（学習中／完了 などのバッジ表示）
- 難易度レベル
- タグ（複数可）
- 削除ボタン（一覧上の操作）

一覧はカテゴリ・タグ・ステータスで絞り込みできるようにする。
各行をクリックすると記事詳細ページへ遷移する。

### 記事データの構造（Markdown frontmatter）
```yaml
title: string
category: string
updatedDate: date
status: "学習中" | "完了"
level: string  # 難易度レベル
tags: string[]
references: { label: string, url: string }[]  # 参考リンク
relatedArticles: string[]  # 関連記事のslug
```
本文はMarkdown本体に記述。画像はfrontmatterまたは本文内で相対パス指定。

### 2. カテゴリ
カテゴリごとに記事を絞り込んで見る一覧ページ。カテゴリ一覧から選ぶと、該当記事のみが表示される。

### 3. メディア
役立つAI関連の参考記事・メディアリンクをまとめたページ（タイトル・URL・簡単なメモを一覧表示）。

### 4. 用語集
AI関連用語を「用語＋説明」の形で自分用に登録・一覧表示する辞書ページ（単語帳形式）。

### 5. FAQ
よく使うAIプロンプト・コマンドなどのナレッジをまとめたページ（質問・回答/内容の形式で一覧表示）。

### 6. 設定
- サイトの見た目（テーマ・配色など）を変更できる設定
- カテゴリ・タグなどのマスター情報を編集できる管理画面

## 実装時の注意
- コンポーネントは必ずmelta-uiのcontract（`design/contracts/components/`）に従うこと
- 独自の配色・シャドウ・装飾は追加しない（`DESIGN.md`の禁止ルールに従う）
- レスポンシブ対応は基本のみでよい（PC閲覧を優先）
